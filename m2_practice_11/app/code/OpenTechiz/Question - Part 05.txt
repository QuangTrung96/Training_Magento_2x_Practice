4. How to validate form data with JS (client side) and PHP (server side) in Magento 2?

CLIENT
- step1: create a test form, the form is created as the attribute for validation.
- step2: accept javascript validation, please run the below script:
		<script type="text/x-magento-init">
		    {
			"#custom-form": {
			    "validation": {}
			}
		    }
		</script>
	or you can make that operation directly in .html like this:
		<form data-mage-init='{"validation": {}}' class="form" id="custom-form" method="post" autocomplete="off">
		or

		<script type="text/javascript">
		require([
		    'jquery',
		    'mage/mage'
		], function($){
		   var dataForm = $('#custom-form');
		   dataForm.mage('validation', {});
		});
		</script>

SERVER

Use Class Zend_Validate

Ví dụ:

if(!\Zend_Validate::is(trim($postData['email']), 'NotEmpty'))
        {
            $error = true;
            $message = "Email can not be empty!";
        }

6.  What  is  VirtualType  in di.xml? Can  we  use  a  data  provider  class  implementation  instead  of  VirtualType  in  XML  ?

Using VirtualType in Magento 2 is really necessary when you are running an virtual store based on Magento 2 platform? On Magento 2 platform, the di.xml supports for two types of node which are a node type and a node virtualtype, meanwhile virtualtype is considered as a perfect method instead of the type. The virtual tape allows inserting different dependencies into the existing classes but not making any change to other classes. With this tutorial, Mageplaza team will lead you to learn how to create and use VirtualType in Magento 2.


Creating VirtualType in Magento 2

#File: app/code/TQT/HelloWorld/etc/di.xml
<config>
    <!-- ... -->
    <virtualType name="ourVirtualTypeName" type="TQT\HelloWorld\Model\Virtualtype">  
    </virtualType>        
</config>
